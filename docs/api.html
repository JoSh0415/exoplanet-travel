REST API for browsing exoplanet destinations and booking travel holidays.

Conventions

  • All responses are JSON (application/json) unless otherwise stated.
  • Pagination uses page (1-indexed) and pageSize.
  • Error responses follow a consistent shape: { "error": { "code": string, "message": string, "details"?: any } }

Exoplanets

Browse exoplanet catalogue and retrieve planet details.

List exoplanets

Returns a paginated list of exoplanets with optional filters.

Typical usage

  • Show a catalogue page (paged list)
  • Filter by vibe or distance range
  • Search by partial planet name using q
query Parameters
page
integer >= 1
Default: 1
Example: page=1

Page number (1-indexed).

pageSize
integer [ 1 .. 100 ]
Default: 20
Example: pageSize=20

Number of results per page (max 100).

q
string
Example: q=gliese

Case-insensitive substring search in planet name.

vibe
string
Example: vibe=Molten Rock

Filter by derived category (e.g., "Molten Rock", "Mysterious").

minDistance
number >= 0

Minimum distance in light years (inclusive).

maxDistance
number >= 0
Example: maxDistance=25

Maximum distance in light years (inclusive).

sort
string
Default: "distance"
Enum: "distance" "discoveryYear" "name"

Field used to sort results.

order
string
Default: "asc"
Enum: "asc" "desc"

Sort order.

Responses

Local development server

http://localhost:3000/api/exoplanets

Deployed server (optional)

https://YOUR-DEPLOYED-URL-HERE/api/exoplanets

  • Admin view of all bookings (no filter)

pageSize
integer [ 1 .. 100 ]
Default: 20
Example: pageSize=20

userId

Create a booking

Notes

  • In the current implementation, userId is provided in the request body.
  • In a future version, userId would be derived from authentication (JWT/session).
